# CLAUDE.md

이 파일은 Claude Code가 프로젝트를 이해하고 코드 리뷰를 수행할 때 참고하는 지침 파일입니다.

## 프로젝트 개요

Vite + React + TypeScript 구성으로 작성된 **압축 파일 재귀 해제 사이트**입니다. (재귀 깊이: `maxDepth = 5`)
한 번에 최대 **20개**의 파일 / 파일당 **2GB** / 총 **5GB**까지 업로드할 수 있으며, 출력 총량 제한은 **20GB**입니다.
지원하는 포맷은 **zip, rar, 7z, tar, gz, bz2, zst** (+ `tar.gz/tar.bz2/tar.zst` 조합)입니다.
OPFS, 스트리밍, 청크 등의 방식으로 빠르고 메모리 부담 없이 쾌적하게 브라우저에서 처리해야 합니다.
출력 및 폭주 방어 등을 방어할 수 있도록 예외 케이스에 철저하게 대응합니다.

## 코드 리뷰 기준

코드 리뷰 시 아래 항목을 반드시 점검하세요.

### 보안

- 하드 코딩된 credentials(비밀번호, API 키 등)은 허용하지 않습니다.
- 민감한 정보는 Keychain 또는 환경변수를 통해 관리합니다.

### 안전성

- 외부 입력값은 항상 검증합니다.
- 배열 접근 시 반드시 범위(bounds) 체크를 수행합니다.
- 강제 언래핑(`!`)은 사용하지 않습니다.
- `eslint-disable...`은 사용하지 않습니다.
- 논리적 오류, 엣지 케이스, 잠재적 문제점 및 메모리 누수 등을 철저히 검토합니다.

### 일반 품질

- 함수는 단일 책임 원칙(SRP)을 따릅니다.
- 프로젝트 개요에 맞게 구현되어 있는지 확인합니다.
- 불필요하거나 부적절한 코드가 없는지 확인합니다.
- 테스트 및 추후 확장이 용이한 구조로 작성합니다.

## 리뷰 명령어

```
/review
/code-review
```

위 명령어를 실행하면 Claude Code가 이 파일의 기준에 따라 코드를 리뷰합니다.
